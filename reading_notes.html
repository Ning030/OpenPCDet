一、项目全貌

1.1 项目摘要

定位 OpenPCDet 是一个清晰、自包含的开源 LiDAR 3D 目标检测框架，由香港中文大学 Shaoshuai Shi 等人开发。

解决的问题 统一了多种主流 3D 检测算法（PointPillar、SECOND、PV-RCNN、VoxelRCNN、CenterPoint、VoxelNeXt、DSVT、BEVFusion 等）在一个模块化的框架中，支持多种数据集（KITTI、Waymo、NuScenes 等）。

核心思想 采用配置驱动的模块化流水线架构——通过 YAML 配置文件定义模型的组件拓扑，所有检测器共享 Detector3DTemplate 基类中的 module_topology 流水线。

项目信息：作者 Shaoshuai Shi，Apache License 2.0，v0.6.0（2022.09），支持 17+ 检测器架构。

build_networks() 方法

def build_networks(self):
    model_info_dict = {
        'module_list': [],
        'num_rawpoint_features': self.dataset.point_feature_encoder.num_point_features,
        'num_point_features': self.dataset.point_feature_encoder.num_point_features,
        'grid_size': self.dataset.grid_size,
        'point_cloud_range': self.dataset.point_cloud_range,
        'voxel_size': self.dataset.voxel_size,
        'depth_downsample_factor': self.dataset.depth_downsample_factor
    }
    for module_name in self.module_topology:
        module, model_info_dict = getattr(self, 'build_%s' % module_name)(
            model_info_dict=model_info_dict
        )
        self.add_module(module_name, module)
    return model_info_dict['module_list']

pcdet/models/detectors/detector3d_template.py:35-50

关键设计模式：model_info_dict 在模块之间传递，每个 build_xxx() 方法会更新该字典中的特征维度等信息（如 num_point_features、num_bev_features），供下一个模块使用。这实现了模块间的信息流传递。

build_xxx() 方法模式

每个构建方法遵循相同模式：

  1. 检查配置中是否存在对应模块 (self.model_cfg.get('VFE', None))
  2. 若不存在则返回 None, model_info_dict
  3. 若存在则从注册表实例化模块（如 vfe.__all__[cfg.VFE.NAME](...)）
  4. 更新 model_info_dict 中的特征维度信息
  5. 将模块添加到 module_list

post_processing() 方法

输入：batch_dict，含 batch_cls_preds、batch_box_preds

输出：pred_dicts（预测框列表）、recall_dict（召回率统计）

逻辑：

  1. 对每个 batch 样本提取分类和框预测
  2. 应用 Sigmoid 归一化
  3. 执行多类别 NMS 或类别无关 NMS
  4. 按分数阈值过滤，保留 top-k 个结果

3.3 VFE 体素特征编码 特征提取

作用：将原始点云编码为体素级特征，是整个流水线的第一步。

调用位置：build_vfe()（detector3d_template.py:52-66）

MeanVFE（最简单的 VFE）

逻辑：对每个体素内的所有点取均值，输出 (num_voxels, C) 的特征张量。

def forward(self, batch_dict):
    voxel_features, voxel_num_points = batch_dict['voxels'], batch_dict['voxel_num_points']
    points_mean = voxel_features[:, :, :].sum(dim=1) / voxel_num_points.view(-1, 1)
    batch_dict['voxel_features'] = points_mean
    return batch_dict

pcdet/models/backbones_3d/vfe/mean_vfe.py:14-31

PillarVFE（PointPillar 专用）

输入：batch_dict['voxels']（shape: [M, max_points, C]）、voxel_coords、voxel_num_points

输出：batch_dict['pillar_features']（shape: [M, num_filters[-1]]）

实现逻辑：

  1. 计算补充特征：点到体素中心的偏移 (f_cluster)、点到柱体中心的偏移 (f_center)
  2. 可选特征：绝对坐标 (USE_ABSLOTE_XYZ，注意是历史拼写) 和距离
  3. 特征拼接：将原始特征 + 补充特征连接
  4. PFN 层堆叠：通过多层 PFNLayer（Linear + BN + ReLU + MaxPool）编码
  5. 掩码处理：对空点位置置零

VFE.USE_ABSLOTE_XYZself.use_absolute_xyz是否使用绝对坐标（legacy 拼写）
VFE.WITH_DISTANCEself.with_distance是否加入距离特征

3.5 BEV 转换模块 维度转换

作用：将 3D 稀疏特征转换为 2D 鸟瞰图（BEV）密集特征。

调用位置：build_map_to_bev_module()（detector3d_template.py:85-95）

HeightCompression（高度压缩）

核心算法：

  1. 将稀疏张量转为密集张量：sparse_tensor.dense() → shape (N, C, D, H, W)
  2. 将高度维度压平到通道维度：(N, C, D, H, W) → (N, C×D, H, W)
def forward(self, batch_dict):
    encoded_spconv_tensor = batch_dict['encoded_spconv_tensor']
    spatial_features = encoded_spconv_tensor.dense()   # → (N, C, D, H, W)
    N, C, D, H, W = spatial_features.shape
    spatial_features = spatial_features.view(N, C * D, H, W)  # 压缩高度
    batch_dict['spatial_features'] = spatial_features
    return batch_dict

BaseBEVBackbone

结构：下采样 + 上采样（类似 FPN）

# 以 PointPillar 配置为例
# LAYER_NUMS: [3, 5, 5]
# LAYER_STRIDES: [2, 2, 2]
# NUM_FILTERS: [64, 128, 256]
# UPSAMPLE_STRIDES: [1, 2, 4]

3.11 损失函数 训练关键

作用：提供 3D 检测任务专用的损失函数。

调用位置：各检测头的 get_loss() 方法中使用。

SigmoidFocalClassificationLoss

公式：FL(p_t) = -alpha_t × (1 - p_t)^gamma × log(p_t)

作用：解决正负样本不平衡问题。gamma 控制难例挖掘强度，alpha 控制正负权重。

公式：当 |x| < beta 时用 L2，否则用 L1。

ResidualCoder（框编码器）

编码方式（7 维）：

xt = (x_gt - x_a) / d_a     # 位置偏移（归一化）
yt = (y_gt - y_a) / d_a
zt = (z_gt - z_a) / h_a
wt = log(w_gt / w_a)        # 尺寸缩放（对数变换）
lt = log(l_gt / l_a)
ht = log(h_gt / h_a)
rt = r_gt - r_a              # 朝向差

其中 d_a = sqrt(l_a^2 + w_a^2) 是对角线长度。
